# Image Forge

一个本地手机端图片生成 app。它在电脑本地启动 Node 服务，手机通过局域网访问网页界面；后端固定调用图片模型接口，前端只负责提示词、参考图、比例、生成张数和历史浏览。

## 启动

```bash
npm start
```

默认地址：

```text
http://localhost:4173
```

手机访问时，确保手机和电脑在同一个 Wi-Fi 下，然后打开：

```text
http://<电脑局域网 IP>:4173
```

macOS 上可以用下面命令查看局域网 IP：

```bash
ipconfig getifaddr en0
```

## 当前支持

- 文生图：不选择参考图时，调用 OpenAI/OpenAI 兼容 `/images/generations`
- 参考图生成：选择参考图时，调用 OpenAI/OpenAI 兼容 `/images/edits`
- 电商模板：内置 `gpt-image2-ecommerce` 的 25 个场景模板，支持自动匹配、手动选择、风格变体、品类提示、UGC 反 AI 感规则
- 灵感库：融合 `awesome-gpt-image-2` 的 Prompt-as-Code 模板思路和 `awesome-gpt-image-2-prompts` 的案例分类，内置 29 个可套用结构化预设
- 灵感搜索：支持按标题、分类、关键词、示例和标签模糊搜索灵感预设
- 灵感排序：人像修图预设和筛选项默认前置，方便直接进入参考图工作流
- 分类策略：电商、广告、海报、信息图、品牌、UI、角色、摄影、人像修图会切换不同策略字段
- 策略默认值：不同灵感会自动填入类型、影调/风格、用途、色调和关键要求，人像修图预设会给出对应修图要求
- 人像修图：内置自然精修、电影感人像、奢华美妆、胶片黑雾、同脸九宫格、棚拍换装、旧照修复、照片风格复刻、社媒头像大片等参考图工作流
- 后台任务：提交后由后端异步生成，页面轮询进度
- 任务持久化：每次提交的任务状态保存到 `data/jobs.json`，刷新页面后任务列表会自动恢复
- 任务删除：任务页可删除单个任务，正在生成的任务删除后不会再写回本地任务和历史列表
- 多图对比：一次生成 1 到 4 张
- 多比例：1:1、2:3、3:2、9:16、16:9、3:4、4:3
- 历史记录：生成图片保存到 `public/generated/`，历史索引保存到 `data/history.json`
- 历史删除：历史页可删除单条历史记录，已生成的图片文件默认保留在 `public/generated/`
- 任务接口：`GET /api/jobs`、`GET /api/jobs/:id`、`DELETE /api/jobs/:id`
- 历史接口：`GET /api/history`、`DELETE /api/history/:id`
- 模板接口：`GET /api/templates`
- 预设接口：`GET /api/presets`
- 调试记录：`GET /api/debug/recent`

## 返回兼容

后端会优先请求 `b64_json` 图片结果，减少二次下载远程图片失败的概率。如果网关不支持该参数，会自动降级为普通 OpenAI 兼容格式；如果上游连接被关闭，会重试后再把详细失败原因写入 `GET /api/debug/recent`。图片请求当前最多等待约 20 分钟。

图片接口请求使用自定义 `undici` dispatcher，响应头等待时间会跟随图片请求超时配置，避免 Node 默认 headers timeout 在长图像任务约 5 分钟时提前截断。

参考图编辑请求会优先使用非 base64 返回和精简提示词；如果上游主动关闭 socket，会自动降级到更保守的输出尺寸继续重试，降低长任务被网关断开的概率。

## 安全说明

API Key 写死在本地后端 `server.js` 中，不会显示在页面上。只建议在可信局域网内使用。后续生产化时，建议改成环境变量、访问密码、HTTPS 或内网 VPN。
